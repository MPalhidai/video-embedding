Michael's Blog

I am building this single blog page to demonstrate how to embed a YouTube video and position elements like a news article.

The first video is an experiment to see if I can embed a video and resize it to my needs. I like to use elements that can be responsive to any size viewport, so I settled on an iframe using width="auto" and height="315". This was a good solution to fit in a bootstrap card but the height is not responsive.

I had a lot of trouble getting the height to fit the size of the div box. Usually, the tactic is to set width : 100% and height : auto but the iframe height really just wanted to be too small. For this blog page, however, I like the look of setting explicit heights and widths. I figure if anyone wants to actually watch the videos, they should at least be large enough to view.

There is no real reason to watch these video besides. I have a personal log of what is in them and I am using as video proof of my #100DaysofCode challenge. If I want to remember anything I know I have done before. My personal records will let me know which video to look at.

These videos are silent because I like to play music or listen to audiobooks while coding. One day I may use them as a vlog or narrate them, but I like how they are for now.

The second video is of my screen while I build this blog page. I would really like for this paragraph to wrap around the bottom of the embedded video to see how floating elements really work. Unfortunately, I'm going to have to think of more to say.

Bootstrap is a wonderful tool. I like the ability to position elements from the html element. I can quickly write the html and visuallize where I want it with a few shorthand commands. Bootstrap has a lot of other useful tools including modals, buttons, cards, and carousels. All of these prebuilt tools gives you an easy basic structure you can custoize to your needs.

If I was to put more time into this page, I would add breaking points for smaller screens. Bootstrap columns (col-md-6) or display (d-sm-flex) shorthands are a beautiful solution to making this page more responsive. The column could set the width of the video players until the viewport was small enough that it would be better to have the video take the full width of the screen. The only reason I'm not going to implement it here, is that I want to show the power of float. Unfortunately, this page looks silly on my phone.
